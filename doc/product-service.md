# Product Service Frontend Guide

This document is a frontend-facing contract for the product APIs.

## Base URL

`/api/wb/v1/products`

## Response Envelope

All JSON endpoints return the shared `ApiResponse` shape.

```json
{
  "message": "Success",
  "statusCode": 200,
  "timestamp": "2026-04-10T10:30:00",
  "data": {}
}
```

For paginated list endpoints, `data` is a paginated object:

```json
{
  "data": [],
  "pagination": {
    "is_first": true,
    "is_last": false,
    "page_size": 10,
    "total_pages": 5,
    "current_page": 1,
    "current_total_elements": 10,
    "total_elements": 50,
    "is_empty": false,
    "has_next": true,
    "has_previous": false,
    "next_page": 2,
    "previous_page": null
  }
}
```

Pagination notes:

- Request query params use 0-based indexing: `page=0` is the first page.
- Response pagination uses 1-based `current_page` for UI display.
- `page_size` is the page size the backend applied.
- `next_page` and `previous_page` are `null` when not available.
- Use `total_pages` and `total_elements` for paginator controls.

## Access Rules

- `GET /api/wb/v1/products` is public.
- `GET /api/wb/v1/products/{id}` is public.
- `GET /api/wb/v1/products/slug/{slug}` is public.
- `POST /api/wb/v1/products` requires `PRODUCT_WRITE`.
- `PUT /api/wb/v1/products/{id}` requires `PRODUCT_WRITE`.
- `DELETE /api/wb/v1/products/{id}` requires `PRODUCT_DELETE`.

The JWT must include the permission claims generated by login. `SELLER` currently receives product write/delete permissions from the seed migration, while `USER` does not.

## Endpoints

### 1. Get all products

`GET /api/wb/v1/products`

Query params:

- `categorySlug` optional
- `filterProductCateType` optional, values: `ALL`, `NO_CATEGORY`
- `search` optional
- `sort` optional
- `page` optional, default `0`
- `size` optional, default `10`

Example:

`/api/wb/v1/products?search=watch&categorySlug=electronics&page=0&size=12`

Pagination behavior:

- `page` is zero-based in the request.
- `size` controls how many items are returned per page.
- If omitted, the backend applies its default pageable settings.
- The response will include the current page metadata inside `data.pagination`.

Frontend usage:

- Product grid
- Search results
- Category landing pages
- Infinite scroll or paginated table

### 2. Get product by id

`GET /api/wb/v1/products/{id}`

Use this for product detail pages when the ID is known.

### 3. Get product by slug

`GET /api/wb/v1/products/slug/{slug}`

Use this for SEO-friendly product detail pages.

### 4. Create product

`POST /api/wb/v1/products`

Permission: `PRODUCT_WRITE`

Request body:

```json
{
  "userId": 10,
  "slug": "wireless-earbuds-51",
  "title": "Wireless Earbuds 51",
  "description": "High quality wireless earbuds for demo storefronts.",
  "price": 35.5,
  "sku": "SKU-00051",
  "currency": "USD",
  "weight": 0.25,
  "length": 10,
  "width": 8,
  "height": 3,
  "taxRate": 0,
  "isActive": true,
  "isFeatured": false,
  "isNew": true,
  "categoryIds": [1, 3],
  "images": [
    {
      "imageUrl": "https://placehold.co/1000x1000/png?text=Wireless+Earbuds+51",
      "altText": "Wireless Earbuds 51 primary image",
      "displayOrder": 1,
      "isPrimary": true
    }
  ],
  "variations": [
    {
      "name": "Color",
      "value": "Black",
      "priceAdjustment": 0,
      "sku": "SKU-00051-BLK",
      "stockQuantity": 100
    }
  ]
}
```

Validation notes:

- `userId` is required.
- `slug`, `title`, and `currency` are required.
- `categoryIds` must not be empty.
- `currency` must be exactly 3 characters.
- `images[0]` becomes the main product image.

### 5. Update product

`PUT /api/wb/v1/products/{id}`

Permission: `PRODUCT_WRITE`

Request body:

```json
{
  "title": "Wireless Earbuds 51 Pro",
  "description": "Updated description",
  "price": 39.99,
  "sku": "SKU-00051",
  "currency": "USD",
  "weight": 0.26,
  "length": 10.2,
  "width": 8.1,
  "height": 3.1,
  "taxRate": 0,
  "isActive": true,
  "isFeatured": true,
  "isNew": false,
  "categoryIds": [1, 2]
}
```

Notes:

- `categoryIds` is required.
- The update endpoint does not currently replace images or variations.
- Frontend should refetch product details after update.

### 6. Delete product

`DELETE /api/wb/v1/products/{id}`

Permission: `PRODUCT_DELETE`

Behavior:

- This is a soft delete.
- `isActive` is set to `false`.

## Related Data For Product Forms

The product form needs category lookup data.

### Get all categories

`GET /api/wb/v1/categories`

Returns:

```json
[
  {
    "id": 1,
    "name": "Electronics",
    "slug": "electronics"
  }
]
```

This endpoint is public and should be used to populate category multi-selects.

## Frontend Integration Notes

- Use `slug` for detail page routes when possible.
- Treat `images[0]` as the primary image.
- Render `categories` as chips or badges.
- Use `isFeatured` and `isNew` for storefront badges.
- Use `isActive` to hide inactive products from public catalog views.
- Keep `price`, `taxRate`, and dimension fields as numbers in the form state, but serialize them as JSON numbers.

## Suggested Frontend Service Methods

- `getProducts(params)`
- `getProductById(id)`
- `getProductBySlug(slug)`
- `createProduct(payload)`
- `updateProduct(id, payload)`
- `deleteProduct(id)`
- `getCategories()`

## Error Handling

Common API messages:

- `Validation failed`
- `Invalid request data`
- `Product not found`
- `Category not found`
- `You do not have permission to perform this action`

Frontend should map:

- `401` to login redirect
- `403` to permission denied UI
- `404` to not found state
- `400` to form validation or request error
